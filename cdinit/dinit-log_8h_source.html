4// Logging for Dinit.
5//
6// The main log function is the variadic template 'log' function:
7//
8// void log(loglevel_t, ...)
9//
10// It takes a list of items comprising a single log message, including strings (C/C++ style), and integers.
11// The loglevel argument determines if the message will actually be logged (according to the configured log
12// level of the log mechanisms).
13//
14// We have two separate log "streams": one for the console/stdout, one for the syslog facility (or log
15// file). Both have a circular buffer. Log messages are appended to the circular buffer (for a syslog
16// stream, the messages are prepended with a syslog priority indicator). Both streams start out inactive
17// (release = true in buffered_log_stream), which means they will buffer messages but not write them.
18//
19// Service start/stop messages for the console stream are formatted differently, with a "visual" flavour.
20// The console stream is treated as informational and in some circumstances messages will be discarded
21// from its buffer with no warning.
22//
23// If a stream buffer becomes full mid-message, the message is discarded and we mark the stream as "message
24// discarded". Once the message at the front of the buffer has been fully output we check for the mark and
25// if set, issue a message informing that log messages have been discarded, before resuming regular output
26// from the buffer. (Because the buffer is full, we can't store the "message discarded" message in it; we
27// temporarily switch to a "special buffer" which just contains the "message discarded" text. Current the
28// special buffer mechanism is used only for this purpose).
29//
30// The console log stream needs to be able to release the console, if a service is waiting to acquire it.
31// This is accomplished by calling flush_for_release() which then completes the output of the current
32// message (if any) and then assigns the console to a waiting service. Once the console is no longer used
33// by any service, the service_set will return it to the log; the time between release and re-acquire is
34// checked and, if it's too large, the entire buffer is discarded; this avoids suddenly and confusingly
35// displaying stale messages.
